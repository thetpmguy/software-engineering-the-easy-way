如果你能读懂这一页，你就能学完这门课。学到最后，你将真正明白软件是怎么运转的，会写出你的第一批程序，发布一个真实的网站，并且能在任何技术对话里从容应对 —— 全部用最朴素的大白话，配上日常生活中的比喻来讲解。

## 🤔 这门课适合谁？

- "我每天都在用各种应用，却**完全不知道这些东西到底是怎么运作的**。"
- "我和工程师*一起*工作（作为经理、设计师、创始人、律师、医生……），我受够了**在会议上只能点头附和**。"
- "我想**转行进入科技行业**，但每一门课都默认我已经懂一些我根本不懂的东西。"
- "我只是**好奇**，我周围的数字世界是怎么搭起来的。"

**没有任何前置要求** —— 我们会解释一切，连"文件（file）"到底是什么都会讲清楚。

## 🎓 你将学到什么

- 计算机、软件、互联网、数据库（database）、云（cloud）和人工智能到底是怎么工作的。
- 如何读写简单的 Python 程序。
- 如何像专业人士一样使用 Git 和 GitHub（你会发布一个属于自己的、真实上线的网站）。
- 真实的软件团队是如何规划、开发、测试并发布产品的。
- 如何与工程师平等对话 —— API、服务器（server）、缺陷（bug）、部署（deploy）、冲刺（sprint），统统给你讲明白。

**我们不会讲的东西：** 高等数学、特定的企业级工具，以及各种炒作。每一个比喻我们都会诚实地告诉你它在哪里"不成立"，而讲人工智能的那一章会坦诚地说明它的局限。

## 📝 测验与实验

- **测验**放在每个模块的文件夹里（`quiz.md`）—— 答案藏在"Show answer（显示答案）"折叠框里，而且每份测验的最后一题永远是*"讲给朋友听"*的挑战。
- **实验**（`lab.md`）是动手环节，而且**完全不用安装任何东西** —— 只用免费的、浏览器里就能跑的工具，手机也友好。知识正是在动手做实验时才真正记牢的。
